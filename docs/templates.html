Templates

Templates provide the opportunity to connect the static content with the environment variables and custom variables with their custom implementation in golang.

It is possible to activate templates on each individual server. Inside the server or hostServers you can specify the templates and provide the following description: "processors" : [{ "name" : "template", "urls" : "urls, as described in providers, for example, **/route.js", "folder": "optional base folder, by default, server's base folder", "params": ["a" | "g" | "p" | ""] }] 

     Note: the first parameter in params is the default option as follows:
     "a" - apply only variable processing
     "g" -apply as well as golang template
     "p" -apply as well as place providers (not finished yet)
     "" - empty, do not apply templates at all

It is important that the file starts with a comment of any style (single line, multiline, HTML comment) and the first word in the template must be "template" + option as described above ("g", "p" or "")

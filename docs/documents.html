Directives

Overview

Every config and template can have the following directives, which are the same as in C++ language.

#if, #else, #elif #endif, #ifdef, #ifndef (also #elifdef, #elifndef) conditional operators

all #if, #ifdef, #ifndef can go together with all #elif, #elifdef, #elifndef, #else

#define #undef define a new variable or undefine

#error generate error

#include "filename" or #include <filename> or #include filename includes other files

{{{expression}}} the expression is immediately inserted in the text upon loading, based on the current availabe variables.

Comments can be started with # at the beginning of line

include directive

#include includes another file into the config or template as if it was directly inserted into the text

If the file name starts with ./ or ../, then the search is made first relative to the present source file.

If the path is absolute (starts with X:\ or X:/ or /), the search is made only in those absolute paths.

The filename can be placed inside double quotes or within < > brackets.

If the filename is placed in double quotes and the path is relative and not starting with ./ or ../, the search is made in the beginning from the current directory of the main config, then in internal directories, then in the current folder.

If the filename is placed in < > brackets and the path is relative and not starting with ./ or ../, the search is made only in internal directories and current folder.

In both cases, if the file is not found, the error is generated.

If the filename is not placed in any quote or brackets, the search is made as if it were in double quotes, but if the file has not been found, it is ignored.
